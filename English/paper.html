A framework to model real-time databases

Rewritten Abstract:

Real-time databases are essential for managing time-sensitive data and executing time-constrained transactions in dynamic environments such as air traffic control, financial systems, and process automation. Traditional database modeling tools and methodologies fall short in addressing these needs due to the absence of support for temporal consistency, dynamic behavior, and timing constraints. This study proposes a comprehensive framework that enhances database modeling by introducing a real-time object-oriented data model and a specialized UML profile (UML-RTDB). The framework supports the design of both data structures and behavior specific to real-time systems, incorporating concepts such as sensor and derived attributes, real-time methods, and local concurrency control. The proposed UML-RTDB profile defines a set of stereotypes (e.g., <<Sensor>>, <<Derived>>, <<Periodic>>) to extend UML 2.1.2 and enable semantic-rich design for real-time applications. To validate the model, we present a case study based on an air traffic control system, demonstrating how real-time classes are translated into an object-relational schema, and how Java code and SQL queries can be automatically generated using the Fujaba CASE tool. The framework addresses limitations of standard modeling practices and offers a path forward for the robust development of real-time database systems.

Keywords:

Real-time data management, Dynamic database behavior, Software modeling tools, Time-sensitive applications, Case study validation
